Scalability & Data Isolation

Independent Scaling Capability

Because each BFF and Service is decoupled, we can scale specific parts of the system based on real-time demand without over-provisioning the entire platform.

*Showing auto-scaling pod count distribution across services.

Multi-Tenancy Strategy

Logical Isolation

Shared infrastructure balances cost and security. We use a "Discriminator Column" strategy.

Database Schema Pattern

TABLE: attendance
-----------------
id (PK)
student_id
date
status
school_id (FK, Index) <-- Discriminator

Query Injection

Every query is intercepted by middleware which appends the tenant scope automatically.

SELECT * FROM exams
WHERE grade = '5'
AND school_id = 101;
